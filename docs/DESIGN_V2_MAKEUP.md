# DESIGN_V2_MAKEUP.md — 보강 시스템 + 카카오 알림톡 UI 설계서

**프로젝트:** Academy Smart Management System (ASMS)
**작성일:** 2026-04-02
**단계:** Phase 2 — 상용화 1차 (보강 + 카카오 알림톡)
**담당:** UI Design Agent
**기반 문서:** DESIGN.md (v1), SCHEMA.md, API_SPEC.md, MVP_SCOPE.md

---

## 0. 설계 원칙

- 기존 DESIGN.md의 색상/타이포/간격/컴포넌트 토큰을 그대로 사용한다
- 신규 페이지는 기존 Layout(Header + Content) 안에 배치한다
- 보강 상태 표현에만 시맨틱 확장 색상을 추가한다 (아래 참조)
- 모바일 우선, 기존 반응형 브레이크포인트 동일 적용

### 보강 전용 시맨틱 색상 (추가분)

```
대기중 (Pending):   #CA8A04 (Yellow 600)  — 기존 Warning과 동일
승인됨 (Approved):  #2563EB (Blue 600)    — 기존 Primary와 동일
완료 (Completed):   #16A34A (Green 600)   — 기존 Success와 동일
거절됨 (Rejected):  #DC2626 (Red 600)     — 기존 Danger와 동일
취소됨 (Cancelled): #64748B (Slate 500)   — 기존 보조 텍스트와 동일
```

기존 3색 체계(Blue, Green/Yellow/Red) 안에서 해결하므로 새로운 색상 추가 없음.

## 2. 컴포넌트 계층 (신규만)

```
MakeupModule/
├── MakeupSlotManager (선생님: 슬롯 생성/관리)
│   ├── SlotCreateForm
│   │   ├── DatePicker (날짜 선택 — shadcn/ui Popover + Calendar)
│   │   ├── TimeRangeInput (시작/종료 시간 — Input x2)
│   │   ├── CapacityInput (정원 — Input type=number)
│   │   ├── RoomInput (교실 — Input, optional)
│   │   └── ClassSelect (대상 수업 — Select)
│   ├── SlotCard (개별 슬롯 표시)
│   │   ├── SlotDateTime (날짜 + 시간 텍스트)
│   │   ├── CapacityBadge (잔여/전체 — Badge)
│   │   ├── EnrolledStudentList (신청 완료 학생 목록)
│   │   └── SlotActions (수정/삭제 — DropdownMenu)
│   └── SlotList (슬롯 목록 — 날짜순 정렬)
│
├── MakeupRequestList (선생님: 대기 중 신청 목록)
│   ├── RequestCard
│   │   ├── StudentInfo (학생명, 학년)
│   │   ├── AbsenceInfo (원래 결석일, 수업명)
│   │   ├── RequestedSlotInfo (신청한 보강 슬롯 날짜/시간)
│   │   ├── RequestNote (요청사항 텍스트)
│   │   └── ApprovalActions
│   │       ├── ApproveButton (승인 — Button Primary)
│   │       ├── RejectButton (거절 — Button Danger)
│   │       └── RescheduleButton (일정변경 — Button Secondary)
│   └── RequestFilters (상태 필터 — Tabs: 전체/대기/승인/거절)
│
├── MakeupCalendar (캘린더 뷰)
│   ├── CalendarHeader (월 네비게이션 — 이전/다음 월)
│   ├── CalendarGrid (7x5~6 그리드)
│   │   └── CalendarDay
│   │       ├── DayNumber (날짜 숫자)
│   │       └── SlotIndicator (해당 일 보강 건수 — 점 또는 숫자)
│   └── DayDetailPanel (클릭 시 해당 날짜 슬롯 목록)
│       └── SlotCard (재사용)
│
├── MakeupRequestForm (학생/학부모: 신청 폼)
│   ├── AbsenceSelector (결석 수업 선택 — Select)
│   ├── AvailableSlotPicker
│   │   └── SlotOption (선택 가능한 슬롯 — Radio 형태)
│   │       ├── SlotDateTime
│   │       └── RemainingCapacity (잔여석 텍스트)
│   ├── RequestNoteInput (요청사항 — Input)
│   └── SubmitButton (신청하기 — Button Primary)
│
├── MakeupStatus (학생/학부모 대시보드용)
│   ├── StatusCard
│   │   ├── SlotDateTime (보강 날짜/시간)
│   │   ├── ClassName (수업명)
│   │   └── StatusBadge
│   │       ├── PENDING  → bg-yellow-100 text-yellow-700
│   │       ├── APPROVED → bg-blue-100 text-blue-700
│   │       ├── COMPLETED → bg-green-100 text-green-700
│   │       ├── REJECTED → bg-red-100 text-red-700
│   │       └── CANCELLED → bg-slate-100 text-slate-600
│   └── StatusList (상태별 정렬 — 대기 > 승인 > 완료)
│
├── MakeupSuggestBanner (출석 페이지용 배너)
│   ├── BannerText ("결석 학생에게 보강을 안내하시겠습니까?")
│   └── BannerAction (보강 슬롯 관리로 이동 — Button)
│
└── NotificationSettings (알림 설정)
    ├── ChannelToggle (SMS / 카카오 알림톡 / 푸시)
    ├── KakaoConfigForm
    │   ├── SenderKeyInput (발신프로필 키 — Input)
    │   ├── TemplateSelector (메시지 템플릿 — Select)
    │   └── TestSendButton (테스트 발송 — Button Secondary)
    ├── ChannelBadge (발송 결과에 채널 표시)
    │   ├── SMS   → border-slate-300 text-slate-600 "SMS"
    │   ├── KAKAO → border-yellow-400 text-yellow-700 "카카오"
    │   └── PUSH  → border-blue-300 text-blue-600 "푸시"
    └── NotificationLog (발송 내역에 채널 컬럼 추가)
```

### 신규 shadcn/ui 컴포넌트 (추가 필요)

## 5. 알림 UI 업데이트 상세

### 5-1. 카카오 알림톡 설정 (admin)

**위치:** `/settings/notifications` (신규 페이지)

**설정 항목:**
- SMS 활성화 토글 (Switch)
- 카카오 알림톡 활성화 토글 (Switch)
- 앱 푸시 활성화 토글 (Switch — Phase 3, 비활성 상태)
- 카카오 발신프로필 키 입력 (Input, type=password로 마스킹)
- 기본 템플릿 선택 (Select: 수업 알림, 보강 알림, 보강 확정 알림)
- 우선 발송 순서 (Select: 카카오 우선 / SMS 우선 / 카카오만 / SMS만)
- 테스트 발송 (Button Secondary)
- 저장 (Button Primary)

### 5-2. 알림 채널 선택 (발송 시)

기존 NotificationPage의 발송 플로우에 채널 선택 추가:

```
기존:  [알림 미리보기] → [알림 발송하기]
변경:  [알림 미리보기] → [채널 선택] → [알림 발송하기]
```

**채널 선택 UI:** 알림 미리보기와 발송 버튼 사이에 인라인 RadioGroup 추가

```
발송 채널: ● 설정 기본값 (카카오 → SMS)  ○ SMS만  ○ 카카오만
```

간결하게 한 줄로 표현. 대부분 기본값 사용 예상.

### 5-3. 발송 결과 채널 표시

기존 NotificationPage의 SendResult 컴포넌트에 ChannelBadge 추가:

```
ChannelBadge 스타일:
  SMS:   border border-slate-300 text-slate-600 bg-white  "SMS"
  카카오: border border-yellow-400 text-yellow-700 bg-white  "카카오"
  푸시:  border border-blue-300 text-blue-600 bg-white  "푸시"
```

뱃지는 `rounded-full px-2 py-0.5 text-xs font-medium` (기존 Badge 토큰과 동일).
배경색은 흰색으로 통일하여 과도한 색상 사용 방지.

### 5-4. 보강 관련 자동 알림

보강 상태 변경 시 자동 알림 발송 (선생님 수동 발송 불필요):

| 이벤트 | 수신자 | 메시지 요약 |

## 7. AI Slop 방지 체크리스트

- [x] 흰 배경에 무조건 그림자 없음 — 모든 카드/패널은 `border border-slate-200` 만 사용
- [x] 불필요한 애니메이션 없음 — 모달 open/close에 최소 fade만, 캘린더 월 전환 시 애니메이션 없음
- [x] 과도한 라운딩 없음 — `rounded-lg` 최대, `rounded-full`은 StatusBadge/ChannelBadge만
- [x] 색상 과다 사용 없음 — 기존 Blue + Green/Yellow/Red 시맨틱만 사용, 카카오 뱃지만 yellow 테두리 추가
- [x] 기존 디자인 시스템과 일관성 유지 — 동일 토큰(간격, 타이포, 버튼 스타일) 사용
- [x] 모바일 반응형 고려 — 모든 신규 컴포넌트에 모바일 레이아웃 정의 완료
- [x] 불필요한 아이콘 남용 없음 — StatusBadge는 텍스트만, 캘린더 슬롯 인디케이터는 작은 점(dot)만
- [x] 과한 padding/margin 없음 — 기존 4px 단위 일관성 유지

| 캘린더 키보드 탐색 | 방향키로 날짜 이동, Enter로 선택 (aria-role="grid") |
| StatusBadge 색상 의존 방지 | 텍스트 라벨 필수 ("대기중", "승인됨" 등), 색상만으로 구분하지 않음 |
| 승인/거절 버튼 확인 | 거절 시 AlertDialog로 확인 ("정말 거절하시겠습니까?") |
| 슬롯 잔여석 0 | 선택 불가 라디오 비활성 + aria-disabled + "마감" 텍스트 |
| 폼 에러 | aria-invalid + aria-describedby로 에러 메시지 연결 |
| 모바일 터치 영역 | 모든 인터랙티브 요소 최소 44x44px |

## 9. 데이터 흐름 요약

### 선생님 플로우

```
1. 슬롯 생성: SlotCreateForm → POST /makeup-slots → SlotList 갱신
2. 신청 확인: MakeupRequestList ← GET /makeup-requests?status=pending
3. 승인/거절: ApprovalActions → PATCH /makeup-requests/:id → Toast + 자동 알림
4. 캘린더 조회: MakeupCalendar ← GET /makeup-slots?month=2026-04
```

### 학생/학부모 플로우

```
1. 결석 확인: AbsenceSelector ← GET /students/:id/absences?unmakeup=true
2. 슬롯 조회: AvailableSlotPicker ← GET /makeup-slots?classId=...&available=true
3. 신청: MakeupRequestForm → POST /makeup-requests → 확인 화면
4. 현황 조회: MakeupStatus ← GET /makeup-requests?studentId=...
```

### 관리자 플로우

```
1. 채널 설정: NotificationSettings → PATCH /settings/notifications
2. 테스트 발송: TestSendButton → POST /settings/notifications/test
```
