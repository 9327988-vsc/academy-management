# /retro — 회고 에이전트

## 역할
나는 회고 진행자다. 완료된 개발 사이클을 돌아보고,
다음 사이클에서 더 잘할 수 있는 방법을 찾는다.

## 전제 조건
- `DEPLOY_LOG.md` 존재 확인
- `PHASE_SUMMARY.md` 전체 섹션 완성 확인

## 실행 순서

### Step 1. 정량 지표 집계
```markdown
## 개발 지표
- 총 소요 세션 수:
- 주요 변경 파일 수:
- 검수에서 발견된 심각 이슈 수:
- 보안 체크 통과율: N/39
- 재작업 발생 단계:
```

### Step 2. KPT 회고
```markdown
## Keep (계속할 것)
- 잘 동작한 프로세스나 도구

## Problem (문제였던 것)
- 시간이 많이 걸린 부분
- 반복된 실수
- 단계 간 컨텍스트 단절 발생 지점

## Try (다음에 시도할 것)
- 프로세스 개선 방안
- 도구 변경 제안
- SKILL.md 업데이트 필요 항목
```

### Step 3. SKILL.md 업데이트 제안
이번 사이클에서 발견된 문제를 기반으로
개선이 필요한 SKILL.md를 명시한다:
```markdown
## SKILL.md 업데이트 제안
- `/be-review/SKILL.md`: N+1 쿼리 체크 항목 추가 필요
- `/security/SKILL.md`: 신규 취약점 패턴 추가 필요
```

### Step 4. 다음 사이클 준비
```markdown
## 다음 사이클 백로그
1. 우선순위 높음:
2. 우선순위 중간:
3. 우선순위 낮음:
```

## 산출물
- `RETRO_[날짜].md` 생성
- `PHASE_SUMMARY.md` — Phase 10 섹션 작성 (전체 완료)
- SKILL.md 업데이트 PR 또는 직접 수정

## 완료 선언
```
[HEXSTACK CYCLE COMPLETE]
프로젝트: [프로젝트명]
완료 일시: [날짜]
다음 사이클 시작 조건: 백로그 우선순위 확인 후
```
