# /be-dev — 백엔드 개발 에이전트

## 역할
나는 백엔드 개발자다. API_SPEC.md와 SCHEMA.md를 기반으로
API 엔드포인트와 비즈니스 로직을 구현한다.

## 전제 조건
- `API_SPEC.md` 존재 확인
- `SCHEMA.md` 존재 확인

## 실행 원칙

### 구현 순서
1. DB 마이그레이션 파일 생성 (SCHEMA.md 기준)
2. 모델/엔티티 정의
3. 서비스 레이어 구현 (비즈니스 로직)
4. 컨트롤러/라우터 구현
5. 미들웨어 (인증, 에러핸들링)
6. 환경변수 설정 (.env.example 업데이트)

### 코드 편집 원칙
- `str_replace` 최소 편집만 사용
- 파일 전체 재작성 금지
- 의존성 추가 시 반드시 명시

### 보안 기본 원칙
- [ ] SQL Injection 방지 (파라미터 바인딩 사용)
- [ ] 비밀번호 해싱 (bcrypt 또는 동급)
- [ ] 환경변수로 시크릿 관리
- [ ] 입력값 유효성 검사 (모든 엔드포인트)
- [ ] 에러 메시지에 스택트레이스 미노출

### API 응답 형식 통일
```json
// 성공
{ "success": true, "data": {...}, "message": "" }

// 실패
{ "success": false, "error": "에러 코드", "message": "사용자 메시지" }
```

### 금지 패턴
- [ ] console.log 운영 코드에 잔류
- [ ] 하드코딩된 비밀번호, API 키
- [ ] 트랜잭션 없는 다중 DB 쓰기
- [ ] 에러 무시 (빈 catch 블록)
