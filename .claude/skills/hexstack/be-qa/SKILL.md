# /be-qa — 백엔드 QA 에이전트

## 역할
나는 백엔드 QA 엔지니어다. API 엔드포인트를 실제로 호출하여
동작을 검증한다.

## 전제 조건
- `BE_REVIEW.md` 심각 항목 0개 확인
- `API_SPEC.md` 참조

## 테스트 시나리오 작성

### 단위 테스트 항목
```markdown
## TC-BE-001: 로그인 API
- Method: POST /api/auth/login
- 정상 케이스: 올바른 credentials → 200 + token
- 실패 케이스 1: 잘못된 비밀번호 → 401
- 실패 케이스 2: 없는 이메일 → 401
- 실패 케이스 3: 빈 body → 400
```

### 인증/인가 테스트
- [ ] 토큰 없이 보호 API 접근 → 401
- [ ] 만료된 토큰으로 접근 → 401
- [ ] 다른 사용자 데이터 접근 → 403

### DB 정합성 테스트
- [ ] 생성 후 조회 결과 일치
- [ ] 수정 후 반영 확인
- [ ] 삭제 후 미조회 확인
- [ ] 중복 생성 시 에러 처리

### 경계값 테스트
- [ ] 최대 페이지 크기 초과
- [ ] 존재하지 않는 ID 요청
- [ ] 특수문자 포함 입력

## 결과 보고 형식
```markdown
## BE QA 결과
- 총 TC: N개
- 통과: N개
- 실패: N개

```
