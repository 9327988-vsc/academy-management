배포 전 체크리스트와 롤백 플랜을 반드시 준비한다.

## 배포 순서

### 1단계: 카나리 배포 (권장)
```
전체 트래픽의 10% → 신규 버전
이상 없으면 → 50% → 100%
```

### 2단계: 헬스체크
```
배포 직후 확인:
- [ ] 메인 페이지 로딩
- [ ] API /health 엔드포인트 응답
- [ ] 로그인 동작
- [ ] 핵심 기능 1개 이상 동작
```

### 3단계: 모니터링 (배포 후 30분)
- 에러율 배포 전 대비 급증 없음
- 응답 시간 정상 범위
- DB 연결 정상

## 롤백 플랜
문제 발생 시 즉시 롤백:
```bash
# 이전 버전으로 롤백 (환경에 맞게 수정)
git revert [커밋 해시]
# DB 롤백 마이그레이션 실행
```

롤백 기준:
- 에러율 1% 이상 급증
- 핵심 기능 동작 불가
- DB 연결 실패

## 배포 금지 시간대
- 금요일 오후 4시 이후
- 공휴일 전날
- 트래픽 피크 시간대
