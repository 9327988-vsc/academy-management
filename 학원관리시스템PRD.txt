## 📌 1. 프로젝트 개요

### 1.1 목적
학원 선생님의 수업 후 반복적인 알림 발송 업무를 자동화하고, 학생-학부모-선생님 간 소통을 효율화하며, AI 기반 개인 맞춤형 학습 지원을 제공하는 통합 학원 관리 시스템

### 1.2 핵심 가치 제안
- **선생님:** 수업 후 알림 발송 시간 90% 절감 (20분 → 2분)
- **학부모:** 실시간 자녀 학습 현황 파악 및 투명한 소통
- **학생:** 24시간 AI 튜터를 통한 자기주도 학습 지원
- **학원:** 체계적인 출결/보강 관리로 행정 효율 증대

### 1.3 타겟 사용자
- **주 사용자:** 중소형 학원 강사 (학생 수 10~100명 규모)
- **부 사용자:** 초등 고학년 ~ 고등학생, 학부모
- **지역:** 대한민국 전역 (한국어 우선 지원)

#### 2.1.3 알림 발송 채널

**우선순위별 발송:**
1. **카카오 알림톡** (1순위)
   - 학원 공식 계정 인증 필요
   - 템플릿 사전 등록 (최초 1회)
   - 발송 성공률 98%+
   - 비용: 건당 약 8-10원

2. **SMS 문자** (2순위, 알림톡 실패 시)
   - NHN Cloud SMS API 활용
   - LMS(장문) 지원 (2,000자)
   - 발송 성공률 99%+
   - 비용: 건당 약 20원 (LMS)

3. **앱 푸시 알림** (3순위, 앱 설치 시)
   - Firebase Cloud Messaging
   - 즉시 알림 + 알림 센터 보관
   - 무료

**발송 로직:**
```
IF 카카오톡 연동 O → 알림톡 발송
  └─ 실패 시 → SMS 발송
ELSE → SMS 직접 발송

AND 앱 설치 O → 푸시 알림 병행 발송
```

### 3.2 데이터베이스 스키마

```sql
-- 사용자 테이블
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(50) NOT NULL,
    role ENUM('teacher', 'student', 'parent') NOT NULL,
    phone VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_email (email),
    INDEX idx_role (role)
);

-- 학원 정보
CREATE TABLE academies (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    address VARCHAR(200),
    phone VARCHAR(20),
    business_number VARCHAR(20),
    owner_id VARCHAR(36),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (owner_id) REFERENCES users(id)
);

-- 강사 정보
CREATE TABLE teachers (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) UNIQUE NOT NULL,
    academy_id VARCHAR(36) NOT NULL,
    subjects JSON, -- ["수학", "영어", "과학"]
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (academy_id) REFERENCES academies(id)
);

-- 학생 정보
CREATE TABLE students (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) UNIQUE NOT NULL,
    academy_id VARCHAR(36) NOT NULL,
    grade VARCHAR(20), -- "초6", "중3", "고1" 등
    school_name VARCHAR(100),
    parent_id VARCHAR(36),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (academy_id) REFERENCES academies(id),
    FOREIGN KEY (parent_id) REFERENCES users(id)
);

-- 학부모-학생 연결 (한 학부모가 여러 자녀 가능)
CREATE TABLE parent_student (
    parent_id VARCHAR(36),
    student_id VARCHAR(36),
    relationship VARCHAR(20), -- "부", "모", "조부모" 등
    PRIMARY KEY (parent_id, student_id),
    FOREIGN KEY (parent_id) REFERENCES users(id),
    FOREIGN KEY (student_id) REFERENCES students(id)
);

-- 수업 정보
CREATE TABLE classes (
    id VARCHAR(36) PRIMARY KEY,
    academy_id VARCHAR(36) NOT NULL,
    name VARCHAR(100) NOT NULL, -- "수학 중3 심화반"
    subject VARCHAR(50) NOT NULL, -- "수학"
    teacher_id VARCHAR(36) NOT NULL,
    schedule JSON, -- [{"day": "월", "start": "17:00", "end": "19:00"}, ...]
    room VARCHAR(50),
    max_students INT DEFAULT 15,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (academy_id) REFERENCES academies(id),
    FOREIGN KEY (teacher_id) REFERENCES teachers(id),
    INDEX idx_subject (subject)
);

-- 수강 정보 (학생-수업 연결)
CREATE TABLE enrollments (
    id VARCHAR(36) PRIMARY KEY,
    student_id VARCHAR(36) NOT NULL,
    class_id VARCHAR(36) NOT NULL,
    enrolled_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status ENUM('active', 'inactive', 'graduated') DEFAULT 'active',
    FOREIGN KEY (student_id) REFERENCES students(id),
    FOREIGN KEY (class_id) REFERENCES classes(id),
    UNIQUE KEY unique_enrollment (student_id, class_id)
);

-- 수업 세션 (실제 진행된 수업 기록)
CREATE TABLE class_sessions (
    id VARCHAR(36) PRIMARY KEY,
    class_id VARCHAR(36) NOT NULL,
    session_date DATE NOT NULL,
    start_time TIME NOT NULL,
    end_time TIME NOT NULL,
    topic VARCHAR(200), -- "이차방정식 활용"
    textbook VARCHAR(100), -- "최상위 수학"
    pages VARCHAR(50), -- "p.124-131"
    key_concepts TEXT, -- "속력/거리 문제 풀이"
    homework TEXT, -- "최상위 수학 p.132-135 (8문제)"
    homework_due_date DATE,
    next_topic VARCHAR(200), -- "이차함수 그래프"
    special_notes TEXT,
    notification_sent BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (class_id) REFERENCES classes(id),
    INDEX idx_session_date (session_date),
    INDEX idx_notification_sent (notification_sent)
);

-- 출결 기록
CREATE TABLE attendance (
    id VARCHAR(36) PRIMARY KEY,
    session_id VARCHAR(36) NOT NULL,
    student_id VARCHAR(36) NOT NULL,
    status ENUM('present', 'absent', 'late', 'excused') NOT NULL,
    check_time TIMESTAMP,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES class_sessions(id),
    FOREIGN KEY (student_id) REFERENCES students(id),
    INDEX idx_status (status),
    UNIQUE KEY unique_attendance (session_id, student_id)
);

-- 보강 슬롯 (선생님이 등록한 보강 가능 시간)
CREATE TABLE makeup_slots (
    id VARCHAR(36) PRIMARY KEY,
    teacher_id VARCHAR(36) NOT NULL,
    class_id VARCHAR(36), -- NULL이면 모든 수업 가능
    slot_date DATE NOT NULL,
    start_time TIME NOT NULL,
    end_time TIME NOT NULL,
    max_students INT DEFAULT 3,
    current_count INT DEFAULT 0,
    status ENUM('available', 'full', 'closed') DEFAULT 'available',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (teacher_id) REFERENCES teachers(id),
    FOREIGN KEY (class_id) REFERENCES classes(id),
    INDEX idx_slot_date (slot_date),
    INDEX idx_status (status)
);

-- 보강 신청
CREATE TABLE makeup_requests (
    id VARCHAR(36) PRIMARY KEY,
    student_id VARCHAR(36) NOT NULL,
    original_session_id VARCHAR(36) NOT NULL, -- 결석한 원래 수업
    slot_id VARCHAR(36), -- 신청한 보강 슬롯
    request_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status ENUM('pending', 'approved', 'rejected', 'completed', 'cancelled') DEFAULT 'pending',
    student_note TEXT,
    teacher_note TEXT,
    approved_at TIMESTAMP,
    completed_at TIMESTAMP,
    FOREIGN KEY (student_id) REFERENCES students(id),
    FOREIGN KEY (original_session_id) REFERENCES class_sessions(id),
    FOREIGN KEY (slot_id) REFERENCES makeup_slots(id),
    INDEX idx_status (status)
);

-- 알림 발송 로그
CREATE TABLE notifications (
    id VARCHAR(36) PRIMARY KEY,
    recipient_id VARCHAR(36) NOT NULL, -- users.id
    recipient_type ENUM('student', 'parent') NOT NULL,
    channel ENUM('kakao', 'sms', 'push', 'email') NOT NULL,
    type ENUM('session_summary', 'absent_notice', 'makeup_reminder', 'general') NOT NULL,
    title VARCHAR(200),
    content TEXT NOT NULL,
    related_session_id VARCHAR(36),
    sent_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    delivery_status ENUM('pending', 'sent', 'failed', 'read') DEFAULT 'pending',
    error_message TEXT,
    FOREIGN KEY (recipient_id) REFERENCES users(id),
    FOREIGN KEY (related_session_id) REFERENCES class_sessions(id),
    INDEX idx_sent_at (sent_at),
    INDEX idx_delivery_status (delivery_status)
);

-- AI 질문 기록
CREATE TABLE ai_questions (
    id VARCHAR(36) PRIMARY KEY,
    student_id VARCHAR(36) NOT NULL,
    subject VARCHAR(50) NOT NULL,
    topic VARCHAR(100),
    problem_type VARCHAR(100),
    difficulty ENUM('easy', 'medium', 'hard'),
    image_url VARCHAR(500),
    problem_text TEXT,
    student_query TEXT,
    ai_response JSON, -- 전체 AI 응답 저장
    solution_steps JSON,
    similar_problems JSON,
    understanding_level ENUM('understood', 'needs_review', 'struggled'),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (student_id) REFERENCES students(id),
    INDEX idx_subject (subject),
    INDEX idx_topic (topic),
    INDEX idx_created_at (created_at)
);

-- AI 학습 분석 (주간 집계)
CREATE TABLE ai_analytics (
    id VARCHAR(36) PRIMARY KEY,
    student_id VARCHAR(36) NOT NULL,
    week_start_date DATE NOT NULL,
    total_questions INT DEFAULT 0,
    subjects JSON, -- {"수학": 10, "영어": 5}
    weak_topics JSON, -- ["이차함수", "확률"]
    strong_topics JSON, -- ["인수분해", "연립방정식"]
    understanding_rate DECIMAL(5,2), -- 0-100%
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (student_id) REFERENCES students(id),
    UNIQUE KEY unique_week_analysis (student_id, week_start_date)
);
```

### 3.4 시스템 흐름도

#### 수업 종료 후 알림 발송 프로세스
```
[선생님] 수업 종료 버튼 클릭
    ↓
[Frontend] 출석/진도/숙제 데이터 입력 화면
    ↓
[선생님] 정보 입력 후 "알림 발송" 클릭
    ↓
[Frontend] POST /api/sessions (세션 생성)
    ↓
[Backend] 데이터베이스에 세션 저장
    ↓
[Backend] 출결 데이터 저장 (attendance 테이블)
    ↓
[Backend] 알림 메시지 생성 (학생별/학부모별)
    ↓
[Backend] 알림 발송 큐에 추가
    ↓
[Background Worker] 비동기 알림 발송
    ├─ 카카오 알림톡 API 호출
    ├─ 실패 시 SMS API 호출
    └─ 앱 설치 시 푸시 알림 발송
    ↓
[Backend] 알림 발송 결과 로그 저장
    ↓
[Frontend] 발송 완료 알림 표시
```

#### AI 문제 풀이 프로세스
```
[학생] 문제 사진 촬영/업로드
    ↓
[Frontend] 이미지 전처리 (회전 보정, 압축)
    ↓
[Frontend] POST /api/ai/questions + 이미지 base64
    ↓
[Backend] S3에 이미지 업로드
    ↓
[Backend] Claude API Vision 호출
    ├─ 이미지 분석
    ├─ 문제 텍스트 추출
    └─ 문제 유형 분류
    ↓
[Backend] Claude API 재호출 (풀이 생성)
    ├─ 단계별 풀이
    ├─ 핵심 개념 설명
    └─ 주의사항 안내
    ↓
[Backend] 응답 파싱 및 DB 저장
    ↓
[Frontend] 풀이 화면 렌더링
    ↓
[학생] "비슷한 문제" 클릭 (선택)
    ↓
[Frontend] POST /api/ai/similar-problems
    ↓
[Backend] Claude API 호출 (유사 문제 3개 생성)
    ↓
[Backend] 유사 문제 저장
    ↓
[Frontend] 연습 문제 목록 표시
```

#### 보강 신청 프로세스
```
[시스템] 결석 감지 (출석 체크 시)
    ↓
[Backend] 보강 대상자 자동 등록
    ↓
[Backend] 학생/학부모에게 보강 신청 링크 발송
    ↓
[학생/학부모] 보강 신청 링크 클릭
    ↓
[Frontend] GET /api/makeup/slots (가능한 슬롯 조회)
    ↓
[Frontend] 보강 일정 선택 화면 표시
    ↓
[학생/학부모] 원하는 시간 선택 후 신청
    ↓
[Frontend] POST /api/makeup/requests
    ↓

    ↓

    ↓

    ↓
[Backend] 결과 알림 발송 (학생/학부모)
    ↓
```

## 🔐 5. 보안 및 개인정보 처리

### 5.1 인증/인가

**JWT 기반 인증:**
```javascript
// Access Token (짧은 유효기간)
{
  userId: "U001",
  role: "teacher",
  academyId: "A001",
  exp: 15분
}

// Refresh Token (긴 유효기간)
{
  userId: "U001",
  tokenVersion: 1,
  exp: 7일
}
```

**역할 기반 접근 제어 (RBAC):**
```
teacher:
  - 자기 수업 관리
  - 출석 체크
  - 알림 발송
  - 보강 승인/거절
  - 학생 학습 분석 조회

student:
  - 자기 정보 조회
  - 숙제 확인
  - AI 질문
  - 보강 신청

parent:
  - 자녀 정보 조회
  - 알림 수신
  - 선생님 문의
  - 자녀 대신 보강 신청
```

### 5.2 데이터 암호화

**전송 중 암호화:**
- HTTPS/TLS 1.3 의무화
- API 통신 모두 암호화

**저장 데이터 암호화:**
- 비밀번호: bcrypt (salt rounds: 12)
- 민감정보: AES-256-GCM

### 5.3 개인정보 처리

**수집하는 개인정보:**
- 필수: 이름, 이메일, 전화번호, 역할
- 선택: 학교명, 학년, 학생-학부모 관계

**보관 기간:**
- 회원 정보: 탈퇴 후 1년
- 수업 기록: 3년 (교육 관계법 준수)
- AI 질문 기록: 1년
- 알림 로그: 6개월

**제3자 제공:**
- Anthropic (AI 분석): 문제 이미지, 학생 학년 정보
- NHN Cloud (SMS): 전화번호, 메시지 내용
